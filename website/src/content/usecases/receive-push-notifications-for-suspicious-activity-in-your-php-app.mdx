## What is suspicious activity monitoring in PHP?

Monitoring suspicious activity in your PHP application means detecting events like failed logins, unusual file access, or SQL injection attempts.

When you receive instant push notifications, you can react quickly to potential security breaches.

For example, if someone tries to log in with the wrong password multiple times, you get an alert and can investigate before damage happens.

## Track events with Tune

Tune is an open source event-tracking tool you can use to send and view security events from any tech stack.

It lets you define events, add avatars, and push notifications instantly when something unusual happens.

![Tune in action](https://tune/images/misc/ops-usage.jpg)

## Setting up Tune

Follow these steps to start tracking suspicious events:

1. Go to [app.tune](https://app.tune) and sign up for a free account.
2. Create a new project in your dashboard.
3. Copy the project API key under Settings.
4. Install the PHP SDK with Composer:
   ```
   composer require tune/tune
   ```
5. Add your API key to your environment as `OPERATIONAL_API_KEY`.

## Code examples for suspicious events

The following examples show how to send different security alerts to Tune.

### 1. Suspicious login attempt
This snippet sends an event when failed logins exceed a threshold.

```php
<?php
require 'vendor/autoload.php';
use Tune\Tune;

$ops = new Tune(getenv('OPERATIONAL_API_KEY'));

$payload = [
    'name' => 'suspicious login attempt',
    'avatar' => 'ðŸ”’',
    'content' => 'Multiple failed login attempts detected for user John Smith'
];

$ops->events->ingest($payload);
```
